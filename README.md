# 【解説】ShellArc Railgun Architecture — 3D幾何学で革新するアニメ制作マネジメント

テクニカルノート（数学証明）はtechnical_note.tex あるいは technical_note.pdfをご参照ください。

本プロジェクトは、2Dアニメーション制作におけるスケジュールと生産管理を、**数学（幾何学）の力で効率化・視覚化するシステム**です。

これまで経験と勘に頼りがちだった制作進行の遅れや進捗の「勢い」を、3次元の空間データに変換することで、**「どこが遅れているか」「どのカットが失速しているか」をリアルタイムで科学的に見つけ出す**ことができます。

---

## なぜ「レールガン（超電磁砲）」なのか？
アニメ制作の「理想的な進捗スケジュール」を3次元空間上の**目に見えないレール（軌道）**として定義します。
そのレールに沿って、各カットの制作スピードを**「モーメンタム（運動量＝勢い）」**として計算し、遅れている部分をピンポイントで加速・誘導していく思想から、この名前がつけられました。

---

## 1. 進行状況を「3D空間」で捉える
プロジェクトのすべての状態を、以下の3つの軸（X・Y・Z）からなる3次元の立方体空間の中に「点」としてプロットします。

* **【X軸】カット数（空間の横幅）**：作品を構成する全カット。
* **【Y軸】作業の工程（空間の奥行き）**：原画、動画、仕上げなど、1つのカットが完成するまでのステップ。
* **【Z軸】スケジュール日数（空間の高さ）**：制作開始から締め切りまでのタイムライン。

クリエイターから成果物（コンポーネント）が提出されるたびに、この3D空間に「点」が打たれていきます。

---

## 2. プロジェクトの健康診断（全体進捗の評価）

### 📊 理想の進捗曲面（Ideal Face）
「この時期には、これくらい終わっているのがベスト」という理想のペースを、3D空間上の**なだらかな美しい曲面（坂道のようなイメージ）**として計算します。

### 🔍 実際のデータとの比較
実際に打たれた提出データの「点」を、理想の曲面と照らし合わせ、今日（当日）の日付データも含めて補正・計算することで、以下の2つの重要指標を弾き出します。

* **平均乖離度（$\mu$）＝ 「全体の遅れ・進み」**
  理想のスケジュールから、プロジェクト全体が何日分ズレているかを一目で示します。
* **進捗のバラつき（$\sigma$）＝ 「特定の工程への偏り」**
  全体的にまんべんなく進んでいるか、あるいは特定のカットや特定の工程（例：動画だけが極端に詰まっているなど）に作業が偏ってボトルネックになっているかを数値化します。

---

## 3. 各カットの「勢い（モーメンタム）」を見極める

アニメ制作のスピードは一定ではありません。昨日まで止まっていたカットが急に動き出したり、順調だったカットが急にストップしたりします。
本システムでは、数学的な補間関数（3次スプライン）を用いて、各カットの進み具合を滑らかな曲線で繋ぎます。

* **生産速度の「見える化」（偏微分）**
  時間が進む（Z軸）のに対して、どれだけのスピードで工程（Y軸）が消化されているかを計算（偏微分）し、そのカットが今持っている**「生体力的な勢い（モーメンタム）」**を算出します。
* **0〜1のスコア化（正規化）**
  すべてのカットの勢いを `0（完全に停止）` から `1（超高速で進行中）` の間でスコア化し、どのカットにブーストが必要かを直感的に比較できるようにします。

### 🗺️ 進捗フロンティア（最前線）
全カットの「今どこまで進んでいるか」の最新位置を一本の線で繋ぎ合わせた**「進捗フロンティア（最前線）」**を可視化します。これにより、戦況の最前線がどこにあるのかがマップ上で一目瞭然になります。

---

## このシステムがもたらすメリット

1. **遅延の早期検知（アラート）**
   「平均乖離度」を監視することで、手遅れになる前にプロジェクト全体のデッドライン超過リスクをいち早く察知します。
2. **ボトルネックの特定（詰まりの発見）**
   「進捗のバラつき」を見ることで、特定のチームや工程で制作が停滞している現象をデータとして早期発見できます。
3. **データ駆動型のリソース配分（ピンポイント補強）**
   「モーメンタム」が低下している（失速している）カットに対して、ピンポイントで人員を補強するなど、無駄のない効率的なマネジメントが可能になります。
