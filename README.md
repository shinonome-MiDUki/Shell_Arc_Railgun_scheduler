# ShellArc Railgun Architecture

本プロジェクトは、2Dアニメーション制作パイプラインにおけるスケジュールおよび生産管理を効率化するために設計された数理モデルです 。進捗状況を3次元の幾何学的な空間として捉え、プロジェクト全体の遅延リスクや各カットの制作の「勢い（モーメンタム）」を定量的に評価します。

進捗の幾何学的軌道（Ideal Face）を「レール」として定義し、各カットの進行速度を「モーメンタム（運動量）」として定量化・加速させる思想に基づいて設計されていることから、軌道誘導型加速装置である「超電磁砲」の名を冠している。

---

## 1. 3次元空間による進捗の定義

プロジェクトの進捗は、以下の3つの軸を持つ離散的な3次元格子空間の中で表現されます

* **X軸（カット数）**: プロジェクトに含まれる全カットを指します 
* **Y軸（コンポーネントの分割精度）**: 各カットを構成する作業要素（原画、動画、仕上げなど）の分割度合いを表します
* **Z軸（プロジェクト総日数）**: 制作開始から終了までの期間（日数）を表します 

提出された成果物は、この空間内の点 $M$ としてプロットされます 

---

## 2. 全体進捗と状態の評価

### 理想的な進捗曲面（Ideal Face $I$）
プロジェクトが最も理想的なペースで進んだ場合の基準となる曲面 $I$ を定義します。この曲面は、数式を用いて滑らかな目標ラインとして設定されます

### 進捗の再配置と評価
実際の提出データ（点 $M$）を、隣接関係の制約を満たしつつ、理想の曲面に最も近くなるような位置へとマッピング（再配置）し、新たな点群 $G$ を生成します 
今日の日付（当日 $T$）のデータを含めた補正を行った後、理想の進捗曲面との乖離を以下の指標で算出します

* **平均乖離度（$\mu$）**: プロジェクト全体の理想からの遅れ、または進み具合を総合的に示します
* **進捗のバラつき（$\sigma$）**: 特定のカットや工程に作業が偏っているか、あるいは全体が均一に進んでいるかの「ばらつき（標準偏差）」を示します 

---

## 3. 各カットの「勢い（モーメンタム）」の評価

制作の進行速度は一定ではありません。各カットの進捗速度のトレンド（勢い）を捉えるため、3次スプライン補間関数を用いて進捗を曲線として滑らかに繋ぎます

* **生産速度の算出（偏微分）**
  Z軸（時間）に対するY軸（作業コンポーネントの進捗）の傾き（偏微分 $\frac{\partial y}{\partial z}$）を計算することで、「そのカットが今どれだけのスピードで進んでいるか」という**モーメンタム（勢い）**を算出します 
* **現在トレンドの補正**
  最新の提出データから今日（当日 $T$）までの期間の勢いも加味して計算されます 
* **正規化（0〜1のスコア化）**
  算出されたモーメンタムは `[0, 1]` の範囲に正規化され、どのカットの勢いが良く、どのカットが停滞しているかを一目で比較できるようにします 

### 進捗フロンティア（$\mathcal{F}$）
全カットの最新の進捗位置を繋ぎ合わせることで、プロジェクト全体の現在の「最前線（進捗フロンティア $\mathcal{F}$）」を可視化し、どこまで制作が到達しているかを明確にします

---

## 期待される効果

1. **遅延の早期検知**: 平均乖離度（$\mu$）の変化から、プロジェクト全体のデッドライン超過リスクをいち早く察知します 
2. **ボトルネックの特定**: 進捗のバラつき（$\sigma$）を見ることで、特定のカットで制作が詰まっている現象を数値的に発見できます
3. **リソースの最適配分**: モーメンタム（$\frac{\partial y}{\partial z}$）が低下している（停滞している）カットに対して、ピンポイントで人員を補強するなどのデータ駆動型のマネジメントが可能になります
